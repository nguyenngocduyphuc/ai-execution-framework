# Contributing to AXF

We welcome contributions! AXF is a community-driven project.

## 🚀 How to Contribute

1.  **Fork the Repo**: Create your own fork.
2.  **Create a Branch**: `git checkout -b feature/my-feature`
3.  **Make Changes**: Follow the AXF philosophy (Verification First, Read Before Write).
4.  **Verify**: Run `python verification/verify_environment.py` and any relevant tests.
5.  **Commit**: Use clear, descriptive commit messages.
6.  **Push**: `git push origin feature/my-feature`
7.  **Pull Request**: Open a PR against the `main` branch.

## 📋 Guidelines

*   **Follow the Contract**: Read `AGENTS.md` before contributing.
*   **Evidence**: Attach evidence to your PR (logs, screenshots, test results).
*   **Scope**: Keep changes scoped to the feature/fix.
*   **Docs**: Update documentation if you change behavior.

## 🐛 Reporting Issues

Use GitHub Issues to report bugs or suggest features.
*   **Bugs**: Include steps to reproduce, expected behavior, and actual behavior.
*   **Features**: Describe the use case and why it matters.

## 🤝 Code of Conduct

Be respectful, constructive, and helpful. We are building a standard together.
